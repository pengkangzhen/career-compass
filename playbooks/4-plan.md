# Playbook 4 — Plan（展开路径 · 可选）

> **这是可选阶段。** 只有当用户从机会矩阵里**自己选定一个方向**后，才进入这里。
> 机会矩阵（`data/opportunities.md`）本身已是完整交付物；不进这一步也完全 OK。

## 进入条件

用户明确说："我想深入 X 方向" / "就选 X 吧"。**不要替用户选**。如果用户没选，就停在矩阵那一步。

## 入口

```bash
uv run career-compass render-strategy   # 生成 strategy.md 骨架
```
然后填实。**覆盖重写**，不要在旧版本上堆叠。

## 选定方向及理由

strategy.md 开头要写明：
- 为什么从矩阵里选这个方向（引 L1-L4 证据）
- **为什么不选第二名的** —— 这个对比比夸第一名更重要
- 这个方向对应矩阵里的 `composite` 评分和 `opens_up`

## 三段时间线

每段都要具体到能判断"做到了没有"。

### 短期 (0-6 月)
- 下一步**具体动作**：学什么、做什么项目、接触谁、投哪类岗
- 优先**低试错成本**步骤（学/试/副项目），除非 reversibility_bias 低
- 明确产出物：一个上线 demo、一份证书、N 次咖啡聊、一个 offer
- 直接用矩阵里该方向的 `first_step` 作为起点

### 中期 (6-18 月)
- 在选定方向上建立**可被外部验证的能力证据**（作品/职级/被引用的工作）
- 写明"18 个月后我是什么样的人，外部怎么看出来"

### 长期 (1-3 年)
- 这条路径的"期权兑现"：通向哪 2-3 个更远位置？（对应矩阵的 `opens_up`）
- 标注哪些分叉点需要届时再决策（不要假装现在能看清 3 年）

## 每步都要写

- **试错成本**：低 / 高（辞职、读博、异地 = 高；学习、副项目 = 低）
- **退出条件**：什么信号出现就说明这步走错了
- **排他性**：做这步排除哪些其它路（对应矩阵的 `costs`）

## 对齐 constraints

- runway 不足 → 短期不能"全职读博"
- risk_appetite low → 短期不能 all-in 没对冲的方向
- （geo/地域已移除——"去硅谷攒经历"等城市选择由用户拿到方向后自行决定，北斗星不约束）

违反任何 constraint = 无效，重写。

## 完成判据

`data/strategy.md` 写完。**强烈建议接着跑 `5-stress-test`** —— 没被反向论证挑战过的路径不算完成。

## 反模式

- ❌ 替用户选方向（你的活是展开，不是拍板）
- ❌ 短期写"提升综合能力"这种无法验证的话
- ❌ 三段时间线只是同一个目标的三个尺寸
- ❌ 三年路径写得像预言（应保留分叉点）
